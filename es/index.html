Nimbus

Tu empleado 24/7.

Una IA que entra en tus apps, envía correos, actualiza tu CRM y ejecuta tareas mientras duermes. Configúrala una vez, gratis para siempre.

Creado por Sunday Johnson en Yoodule.

Dos formas de usar Nimbus

Self-host

Gratis para siempre

  • Se ejecuta en tu portátil, tu servidor, tu CI
  • Instalación con un solo comando en macOS, Linux y Windows
  • Código fuente completo, licencia BUSL-1.1
  • Lanzamientos verificados con SHA256 y atestados con SLSA
Obtener el instalador

Nimbus Cloud

Gestionado · Operado por Yoodule

  • Nosotros aprovisionamos, parchamos y monitorizamos 24/7
  • Disponibilidad de grado producción y respuesta a incidentes
  • Ayuda para migrar desde self-host cuando estés listo
  • El mismo Nimbus, los mismos servidores MCP, sin carga operativa

Self-host Nimbus es y siempre será gratis. Cloud es para equipos que prefieren pagarnos a operarlo en lugar de hacerlo ellos mismos.

Por qué Nimbus

No quieres otra herramienta. Quieres un empleado — uno que se encargue del trabajo repetitivo, aprenda cómo trabajas y esté ahí a las 2 de la mañana cuando un cliente escribe. Nimbus es ese empleado. Se conecta a las apps que ya usas, sigue los pasos que tú darías y se vuelve más inteligente cada vez que se ejecuta.
Trabaja mientras duermes Programa una tarea a cualquier hora — seguimientos diarios, informes semanales, comprobaciones por hora. Nimbus las ejecuta y te avisa cuando termina.
Usa las apps que ya pagas Notion, Gmail, tu CRM, tu calendario. Nimbus inicia sesión y actúa en ellas — sin copiar y pegar, sin exportaciones.
Pregunta antes de hacer algo arriesgado Envía un borrador en vez de un correo, te muestra un plan antes de borrar una fila. Tú siempre tienes el control.
Se repara solo cuando algo se rompe Una contraseña rotada, un servicio reiniciado, una app actualizada — Nimbus lo detecta y lo arregla. No te despiertan las alertas.

Qué puede hacer

📨 Enviar correos de seguimiento

Dile a Nimbus "envía un correo a todos los que descargaron el whitepaper la semana pasada pero no reservaron una llamada". Escribe un borrador, espera tu aprobación y lo envía.

📊 Crear informes con tus datos

"Saca los tratos cerrados del mes pasado, agrúpalos por sector y pon un gráfico en la página compartida de Notion." Listo por la mañana. Sin pelearte con hojas de cálculo.

🗓️ Clasificar tu bandeja de entrada

Cada mañana, Nimbus etiqueta los correos nuevos, redacta respuestas para los evidentes y marca el resto en Slack. Tú solo ves los que te necesitan.

🔁 Ejecutar workflows en un horario

"Todos los viernes a las 4pm, publica los principales logros de la semana en #team-channel y envíame un resumen por correo." Nimbus lo ejecuta para siempre, sin mantenimiento.

🧠 Recuerda cómo trabajas

Cuanto más usas Nimbus, más aprende. Sabe qué correos siempre archivas, qué informes importan y qué pasos puede saltarse.

🛡️ Nunca envía nada que no hayas aprobado

Nimbus primero redacta, tú firmas. No puede enviar un correo, borrar una fila ni publicar un mensaje sin tu clic. Pista de auditoría completa, cada acción.

Dónde encaja en tu negocio

Para un fundador que hace de todo

Deja de pasar los domingos escribiendo correos de seguimiento y actualizando tu CRM. Entrega el trabajo repetitivo a Nimbus y usa el tiempo para lo que solo tú puedes hacer.

nimbus start — configúralo una vez en tu portátil, gratis para siempre.

Para un equipo pequeño

Dale a cada persona del equipo su propio empleado Nimbus. Comparten contexto, recuerdan las preferencias de cada uno y nunca pierden una nota de cliente.

Cada acción queda firmada, auditada y es reversible.

Para una empresa que necesita control

Ejecuta Nimbus dentro de tu propia infraestructura, con tus claves y tus registros de auditoría. O déjanoslo a nosotros y lo operamos. El mismo empleado de las dos formas.

La pasarela semántica (mcp.json)

Para los curiosos: Nimbus se basa en un pequeño archivo de configuración que lista todas las apps que tu empleado puede usar. Probablemente no necesites tocarlo — Nimbus viene con valores predeterminados razonables.

Nimbus agrega todos tus servidores MCP basados en stdio y HTTP en una única interfaz. Cuando un modelo de IA solicita una capacidad, Nimbus descubre y ejecuta dinámicamente la herramienta adecuada usando embeddings vectoriales almacenados en Qdrant.

Configura tus servidores activos modificando mcp.json en tu directorio home de Nimbus:

{
  "mcpServers": {
    "playwright-mcp": {
      "url": "http://localhost:3100/mcp",
      "transport": "streamable-http"
    },
    "notion-mcp": {
      "url": "https://mcp.notion.com/mcp",
      "transport": "streamable-http",
      "auth": "oauth"
    },
    "linkedin-mcp": {
      "command": "uv",
      "args": ["run", "-m", "linkedin_mcp_server", "--transport", "stdio"]
    }
  }
}

La pasarela expone cuatro endpoints principales a los modelos (bajo el namespace nimbus-utility-mcp):

  1. list_servers — Lista todos los servidores MCP configurados con su estado de conexión y número de herramientas.
  2. find_tools — Busca semánticamente entre todos los servidores. Tu modelo pide "explorar linkedin" y recibe solo los esquemas de herramientas relevantes.
  3. execute_tool — Ejecuta cualquier herramienta en cualquier servidor registrado de forma dinámica.
  4. chain_tools — Encadena varios pasos secuencialmente (p. ej. buscar → leer → enviar) en una sola llamada a la API.

Referencia del CLI

Para el usuario técnico: cada comando que Nimbus entiende. La mayoría de personas solo necesitan nimbus start y nimbus doctor — el resto es para cuando algo va mal.

El CLI nimbus controla el ciclo de vida de tu espacio de trabajo local en contenedores.

Comando Acción
nimbus start Inicia los servicios. Aprovisiona y arranca todos los contenedores Docker (postgres, redis, qdrant, gateway).
nimbus start --build Reconstruye e inicia. Reconstruye las imágenes Docker desde el código actual e inicia los servicios limpiamente.
nimbus start --gateway Modo headless. Ejecuta solo el contenedor de la pasarela — omite la pila completa para CI o uso solo MCP remoto.
nimbus start --fresh Borra los volúmenes de datos. Regenera las credenciales de Postgres/Redis y reinicializa el directorio de datos.
nimbus stop Apagado. Termina de forma segura todos los servicios de contenedores y libera los puertos del sistema.
nimbus upgrade Actualiza. Descarga los últimos binarios y actualiza la configuración del wrapper.
nimbus update Actualiza el CLI. Refresca la imagen del contenedor de la pasarela (junto con --gateway).
nimbus uninstall Desinstala. Elimina el CLI de Nimbus, los contenedores y la configuración.
nimbus uninstall --keep-data Desinstala pero conserva las bases de datos, claves y configuración.
nimbus uninstall --purge Desinstala y borra todo sin pedir confirmación.
nimbus doctor Diagnostica. Comprueba Docker, desfase de versiones, salud de los contenedores, actualidad de actualizaciones, deriva de credenciales.
nimbus recover-pg-role Repara. Migra el volumen nimbus_postgres_data a las credenciales actuales del .env.
nimbus recover-redis-password Repara. Rota la contraseña del volumen Redis para que coincida con el .env actual.
nimbus dashboard install Instala el companion opcional Agent Dashboard.
nimbus dashboard uninstall Elimina el contenedor del Dashboard y su bloque de servicio.
nimbus dashboard status Muestra si el Dashboard está habilitado y el estado de su contenedor.
nimbus config <subcomando> Gestiona ~/.nimbus/.env (list, get, set, unset).
nimbus mcp <subcomando> Gestiona ~/.nimbus/mcp.json (list, get, set, remove).
nimbus chat "prompt" Envía un prompt al dashboard y transmite la respuesta.
nimbus env-init Rellena los secretos gestionados por el CLI en ~/.nimbus/.env (idempotente).

Consulta nimbus --help para la referencia completa, o la página de descarga para instrucciones de instalación específicas por plataforma.
