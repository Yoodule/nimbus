Votre employé disponible 24 h/24, 7 j/7.

Pourquoi Nimbus

Vous ne voulez pas un outil de plus. Vous voulez un employé — un qui s'occupe des tâches répétitives, apprend votre façon de travailler et est là à 2 h du matin quand un client envoie un e-mail. Nimbus est cet employé. Il se branche sur les applications que vous utilisez déjà, suit les étapes que vous feriez et devient plus intelligent à chaque exécution.
Travaille pendant que vous dormez Planifiez une tâche à n'importe quel moment — relances quotidiennes, rapports hebdomadaires, vérifications horaires. Nimbus les exécute et vous prévient quand c'est terminé.
Utilise les apps pour lesquelles vous payez déjà Notion, Gmail, votre CRM, votre calendrier. Nimbus se connecte et agit dessus — pas de copier-coller, pas d'exports.
Demande avant toute action risquée Envoie un brouillon au lieu d'un e-mail, vous montre un plan avant de supprimer une ligne. Vous gardez toujours le contrôle.
Se répare tout seul quand quelque chose casse Un mot de passe changé, un service redémarré, une application mise à jour — Nimbus le détecte et le corrige. Vous n'êtes pas sollicité.

Ce qu'il peut faire

📨 Envoyer des e-mails de relance

Dites à Nimbus « envoie un e-mail à tous ceux qui ont téléchargé le livre blanc la semaine dernière mais n'ont pas réservé d'appel ». Il rédige un brouillon, attend votre validation, puis envoie.

📊 Construire des rapports à partir de vos données

« Extrais les transactions conclues le mois dernier, regroupe-les par secteur et mets un graphique sur la page Notion partagée ». Terminé au matin. Pas de bricolage de tableur.

🗓️ Trier votre boîte de réception

Chaque matin, Nimbus étiquette les nouveaux e-mails, rédige des réponses pour les évidents et signale le reste dans Slack. Vous ne voyez que ceux qui ont besoin de vous.

🔁 Exécuter des workflows selon un planning

« Chaque vendredi à 16 h, poste les principaux résultats de la semaine dans #team-channel et envoie-moi un résumé par e-mail ». Nimbus l'exécute pour toujours, sans maintenance.

🧠 Se souvient de votre façon de travailler

Plus vous utilisez Nimbus, plus il apprend. Il sait quels e-mails vous archivez toujours, quels rapports comptent et quelles étapes il peut sauter.

🛡️ N'envoie jamais rien sans votre accord

Nimbus rédige d'abord, vous validez. Il ne peut pas envoyer un e-mail, supprimer une ligne ni publier un message sans votre clic. Journal d'audit complet, chaque action.

Où il s'intègre dans votre activité

Pour un fondateur qui fait tout

Arrêtez de passer vos dimanches à rédiger des e-mails de relance et à mettre à jour votre CRM. Confiez le travail répétitif à Nimbus et consacrez votre temps à ce que vous seul pouvez faire.

nimbus start — configurez une fois sur votre ordinateur portable, gratuit pour toujours.

Pour une petite équipe

Donnez à chaque membre de l'équipe son propre employé Nimbus. Ils partagent le contexte, mémorisent les préférences de chacun et ne perdent jamais une note client.

Chaque action est signée, auditée et réversible.

Pour une entreprise qui a besoin de contrôle

Exécutez Nimbus dans votre propre infrastructure, avec vos propres clés et vos propres journaux d'audit. Ou confiez-le-nous et nous l'opérons. C'est le même employé dans les deux cas.

La passerelle sémantique (mcp.json)

Pour les curieux : Nimbus repose sur un petit fichier de configuration qui liste toutes les applications que votre employé peut utiliser. Vous n'aurez probablement pas besoin d'y toucher — Nimbus est livré avec des valeurs par défaut raisonnables.

Nimbus regroupe tous vos serveurs MCP en stdio et HTTP derrière une interface unique. Lorsqu'un modèle d'IA demande une capacité, Nimbus découvre et exécute dynamiquement l'outil approprié à l'aide d'embeddings vectoriels stockés dans Qdrant.

Configurez vos serveurs actifs en modifiant mcp.json dans votre répertoire home Nimbus :

{
  "mcpServers": {
    "playwright-mcp": {
      "url": "http://localhost:3100/mcp",
      "transport": "streamable-http"
    },
    "notion-mcp": {
      "url": "https://mcp.notion.com/mcp",
      "transport": "streamable-http",
      "auth": "oauth"
    },
    "linkedin-mcp": {
      "command": "uv",
      "args": ["run", "-m", "linkedin_mcp_server", "--transport", "stdio"]
    }
  }
}

La passerelle expose quatre endpoints principaux aux modèles (sous le namespace nimbus-utility-mcp) :

  1. list_servers — Liste tous les serveurs MCP configurés avec leur état de connexion et le nombre d'outils.
  2. find_tools — Effectue une recherche sémantique sur tous les serveurs. Votre modèle demande « parcourir linkedin » et ne reçoit que les schémas d'outils pertinents.
  3. execute_tool — Exécute dynamiquement n'importe quel outil sur n'importe quel serveur enregistré.
  4. chain_tools — Enchaîne plusieurs étapes séquentiellement (par ex. rechercher → lire → envoyer) en un seul appel API.

Référence du CLI

Pour l'utilisateur technique : toutes les commandes que Nimbus comprend. La plupart des gens n'ont besoin que de nimbus start et nimbus doctor — le reste sert quand quelque chose va de travers.

Le CLI nimbus contrôle le cycle de vie de votre espace de travail local en conteneurs.

Commande Action
nimbus start Démarre les services. Provisionne et lance tous les conteneurs Docker (postgres, redis, qdrant, gateway).
nimbus start --build Reconstruit et démarre. Reconstruit les images Docker depuis la source actuelle et démarre les services proprement.
nimbus start --gateway Mode headless. Exécute uniquement le conteneur de la passerelle — ignore la pile complète pour la CI ou un usage MCP distant uniquement.
nimbus start --fresh Efface les volumes de données. Régénère les identifiants Postgres/Redis et réinitialise le répertoire de données.
nimbus stop Arrête. Termine proprement tous les services conteneurisés et libère les ports système.
nimbus upgrade Met à niveau. Récupère les derniers binaires et met à jour la configuration du wrapper.
nimbus update Met à jour le CLI. Rafraîchit l'image du conteneur de la passerelle (avec --gateway).
nimbus uninstall Désinstalle. Supprime le CLI Nimbus, les conteneurs et la configuration.
nimbus uninstall --keep-data Désinstalle mais conserve les bases de données, les clés et la configuration.
nimbus uninstall --purge Désinstalle et efface tout sans confirmation.
nimbus doctor Diagnostique. Vérifie Docker, les écarts de version, la santé des conteneurs, la fraîcheur des mises à jour, la dérive des identifiants.
nimbus recover-pg-role Répare. Migre le volume nimbus_postgres_data vers les identifiants actuels du .env.
nimbus recover-redis-password Répare. Fait tourner le mot de passe du volume Redis pour qu'il corresponde au .env actuel.
nimbus dashboard install Installe le companion Agent Dashboard optionnel.
nimbus dashboard uninstall Supprime le conteneur du Dashboard et son bloc de service.
nimbus dashboard status Indique si le Dashboard est activé et l'état de son conteneur.
nimbus config <sous-commande> Gère ~/.nimbus/.env (list, get, set, unset).
nimbus mcp <sous-commande> Gère ~/.nimbus/mcp.json (list, get, set, remove).
nimbus chat "prompt" Envoie un prompt au dashboard et diffuse la réponse en streaming.
nimbus env-init Renseigne les secrets gérés par le CLI dans ~/.nimbus/.env (idempotent).

Consultez nimbus --help pour la référence complète, ou la page de téléchargement pour les instructions spécifiques à chaque plateforme.
